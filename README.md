# 3eyes - 人员靠近检测警示系统

基于 YOLOv8 的实时摄像头人员检测系统，当检测到有人从远处走近时自动弹出桌面警示窗口。

## 功能特性

- **实时人员检测** — 使用 YOLOv8n 模型，CPU 即可流畅运行
- **靠近状态追踪** — 通过检测框面积变化判断人员是否从远到近
- **桌面弹窗警示** — 灰色风格 Tkinter 弹窗，支持系统托盘隐藏后台运行
- **GUI 配置界面** — 可视化调整摄像头、检测阈值、弹窗参数
- **配置持久化** — 配置保存到 JSON 文件，重启自动加载


## 快速开始

### 环境要求

- Python 3.8+
- Windows 系统
- 摄像头设备

### 安装与运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 运行程序
python app.py
```

或双击 `run.bat` 一键启动。

### 按键操作

| 操作 | 说明 |
|------|------|
| 点击「开始检测」 | 保存配置并进入后台托盘模式 |
| 点击「保存」 | 保存配置并应用参数到当前预览窗口 |
| 系统托盘右键 → 打开配置 | 恢复配置窗口 |
| 系统托盘右键 → 退出 | 完全退出程序 |

## 设计思路

### 技术选型

| 模块 | 技术 | 选型理由 |
|------|------|----------|
| 摄像头采集 | OpenCV (cv2) | 跨平台、性能优异、支持 DirectShow/Media Foundation 等多种后端 |
| 人员检测 | Ultralytics YOLOv8n | Nano 模型仅 6MB，CPU 可跑 100+ FPS；COCO 预训练已含 person 类(类别0)，开箱即用 |
| GUI 弹窗 | Tkinter (Python 内置) | 零额外依赖、轻量级，配合 `overrideredirect` 可实现无边框自定义窗口 |
| 系统托盘 | pystray + Pillow | 支持后台常驻，右键菜单控制显隐和退出 |
| 配置管理 | JSON 文件持久化 | 无需数据库，配置文件放在 `%APPDATA%/3eyes/` 下，启动自动加载 |

### 为什么用检测框面积而非深度相机

深度相机或测距传感器虽然精度更高，但需要额外硬件。本项目采用**单目 RGB 摄像头**，利用"近大远小"的透视原理——当人靠近摄像头时，YOLO 检测框面积会逐渐增大。通过追踪面积变化趋势即可判断靠近行为，无需额外设备投入。

### 靠近判定状态机

```
                检测到人且面积小
  [IDLE]  ──────────────────────────>  [FAR]
                                         │
                    面积增长超过阈值       │
                    且面积达到中等        │
                                         ▼
                                    [APPROACHING]
                                         │
                    面积进一步增大        │
                    超过"近"阈值          │
                                         ▼
                                      [NEAR]  ───> 触发弹窗警示
                                         │
                    人员消失              │
                                         ▼
                                      [IDLE]
```

**状态说明**：
- **IDLE**: 未检测到人员，重置所有追踪数据
- **FAR**: 检测框面积 < 画面 10%，记录历史最小面积作为比较基准
- **APPROACHING**: 面积相比基准增长超过 50%，且连续 N 帧确认（默认 5 帧）
- **NEAR**: 面积 > 20%，触发弹窗，进入冷却期

### 防抖与去重策略

| 机制 | 实现 | 目的 |
|------|------|------|
| 连续帧确认 | 需连续 N 帧满足条件才切换状态，单帧不满足则计数递减 | 避免 YOLO 单帧误检或抖动导致误报 |
| 移动平均滤波 | 维护面积历史队列（默认 5 帧），取均值平滑波动 | 过滤检测框尺寸的帧间跳变 |
| 冷却时间 | 触发弹窗后 N 秒内不重复弹窗（默认 10 秒） | 防止同一个人持续在近处导致弹窗刷屏 |

### GUI 设计要点

- **单例 Tk root**：整个应用只有一个 `tk.Tk()` 实例，弹窗使用 `Toplevel` 挂载，避免多窗口竞争
- **线程安全**：YOLO 检测在后台线程执行，通过 `queue.Queue` 将帧传递到主线程渲染，通过 `threading.Lock` 保护检测快照读写
- **打印重定向**：`print()` 全部重定向到 logging，同时写入日志文件和控制台 UI 面板，便于排查问题
- **双模式运行**：窗口模式可实时预览检测画面；托盘模式为后台静默运行，摄像头按检测间隔开关以节省 DSHOW 资源

### 弹窗设计（灰色风格）

选择灰色系配色而非传统红色/橙色警示色的原因：避免在办公环境中引起不必要的注意或恐慌，低调但不失提醒效果。

- 背景主色 `#f5f5f5`（浅灰白）、标题栏 `#757575`（中灰）、按钮 `#9e9e9e`
- 无边框窗口 (`overrideredirect`)，始终置顶 (`-topmost`)
- 从屏幕右下角向上滑入动画，定位到鼠标所在显示器
- 支持手动关闭或自动倒计时消失

## 配置参数

所有参数可通过 GUI 调整，保存后自动持久化：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 摄像头 ID | 0 | 摄像头设备编号 |
| 分辨率 | 640x480 | 处理帧尺寸 |
| 检测帧率 | 10 FPS | YOLO 检测间隔 |
| 置信度阈值 | 0.5 | YOLO 检测下限 |
| 近距阈值 | 20% | 检测框占比达到此值触发警示 |
| 远距阈值 | 10% | 检测框低于此值判定为远处 |
| 确认帧数 | 5 | 连续确认帧数 |
| 增长比 | 1.5x | 面积增长倍数 |
| 冷却时间 | 10s | 触发后多久不重复弹窗 |
